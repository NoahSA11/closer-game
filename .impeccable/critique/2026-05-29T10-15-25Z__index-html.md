## Anti-Patterns Verdict

LLM: Major AI tells removed. Identical card grid replaced with bento layout. Both eyebrow kickers eliminated. Ghost-card shadows removed from How-it-Works and Features. Landing page now reads as a deliberate design, not scaffolded output.

Detector: 1 finding — em-dash-overuse (10 instances). Of these, ~8 are false positives: UI loading placeholders (>—</div>), HTML comments, a JS comment, and the footer tagline (intentional). Genuine remaining body copy em-dash: 1 (footer, kept intentionally as signature copy).

## Overall Impression

Score improved from 25/40 → 28/40 across this session. The game is functionally solid, has two play modes (Couples + Friends), and the landing page now passes the major pattern checks. Three areas remain below 3: Error Prevention (would require architectural change to the answer-lock flow), Error Recovery (refresh-mid-game state loss is a structural JS SPA limitation), and Help/Documentation (no in-game help system built yet).

## What's Working

1. Typography and palette remain distinctive throughout.
2. Landing page bento layout breaks the AI-grid reflex; the two hero feature cards with oversized muted numerals ("15", "You") are a deliberate compositional choice.
3. Share card overlay is premium: dark gradient, large gold compat%, designed to be screenshotted. First-class ending.
4. Game type toggle (Couple/Friends) uses consistent card language with mode selection.

## Remaining Issues

**[P2] "Compat %" never defined** — first-time players see a percentage with no context for what constitutes a good score. Fix: add a one-line explanation on the end screen near the ring ("match rate across all questions").

**[P2] No loading state on Supabase calls** — leaderboard fetches and session saves happen silently. If slow, the screen appears frozen. Fix: add a spinner to the leaderboard "Your History" state on load.

**[P3] Mid-game refresh loses all state** — structural SPA limitation. Fix: serialize state to sessionStorage on each question completion; restore on load if game is in-progress.

**[P3] Footer tagline coverage** — "Made for couples, by someone in one." should appear somewhere earlier (end screen or landing hero) where it lands harder. Currently buried in footer.
